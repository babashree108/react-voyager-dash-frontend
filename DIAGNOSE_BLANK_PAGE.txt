-----------------------------

  - Port binding errors
  - Java exceptions


STEP 4: Test API directly
--------------------------

STEP 5: Check browser console
------------------------------
1. Open http://localhost
2. Press F12 (Developer Tools)
3. Go to Console tab
4. Look for errors:
   - Network errors (API calls failing)
   - JavaScript errors
   - 404 errors


STEP 6: Check Network tab
--------------------------
1. F12 ‚Üí Network tab
2. Refresh page
3. Look at API calls:
   - Are they being made?
   - What's the status code?

   - 404 = Not found
   - 500 = Server error


QUICK FIXES:
============

FIX 1: Restart services inside container
-----------------------------------------
docker exec nxtclass-local supervisorctl restart all


FIX 2: Check MySQL is running
------------------------------
docker exec nxtclass-local mysql -u nxtclass_user -pnxtclass_pass_2024 -e "SHOW DATABASES;"

Should show: nxtclass_db


FIX 3: Restart entire container
--------------------------------
docker-compose -f docker-compose.local.yml restart


FIX 4: View all logs
--------------------
docker-compose -f docker-compose.local.yml logs -f

Watch for errors as services start


COMMON ISSUES:
==============

Issue 1: Backend not started
Solution: docker exec nxtclass-local supervisorctl restart backend

Issue 2: MySQL not ready
Solution: docker exec nxtclass-local supervisorctl restart mysql
          docker exec nxtclass-local supervisorctl restart backend

Issue 3: Port 8080 conflict inside container
Solution: docker-compose -f docker-compose.local.yml down
          docker-compose -f docker-compose.local.yml up -d
